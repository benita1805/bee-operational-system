1. OVERVIEW
-----------
The Bee Operational System is a comprehensive platform designed for 
managing beekeeping operations. It enables administrators and 
beekeepers to track hives, manage farmer relationships, and maintain 
geographic data for pollination services.

The system consists of a cross-platform mobile application (Frontend) 
and a robust REST API with offline synchronization capabilities (Backend).


2. KEY FEATURES
---------------
- Farmers Management:
  * Register farmers with their location and crop information.
  * Search and list registered farmers.
  * Integrated map view for geographic visualization.
- Hive Management:
  * Create and track individual bee hives.
  * Record placement dates, harvest expectations, and health notes.
  * Upload and manage media (photos/data) related to specific hives.
- Communication:
  * Real-time chat functionality for collaboration with farmers.
- Offline Capability:
  * Local data persistence for remote field work.
  * Bi-directional synchronization with conflict resolution (Last-Write-Wins).
- Authentication & Security:
  * Phone-based OTP (One-Time Password) login for secure access.
  * Role-based access and Row Level Security (RLS) on the database.


3. FILE STRUCTURE & COMPONENTS
------------------------------

FRONTEND (Root: ./frontend)
---------------------------
The frontend is built using React Native and Expo.

- /app: Contains Expo Router file-based routing.
- /src/screens: UI Layer
  * DashboardScreen.js: Overview and metrics.
  * MapScreen.js: Geographic view of assets and farmers.
  * /Farmers:
    - FarmerRegisterScreen.js: Registration flow.
    - FindFarmersScreen.js: Directory and search.
    - FarmerChatScreen.js: Communication interface.
  * /Hives:
    - MyHivesScreen.js: List of managed hives.
    - HiveFormScreen.js: Hive creation and editing.
    - HiveDetailScreen.js: Detailed logs and media for a hive.
- /src/services: Data Layer
  * apiClient.ts: Axios configuration for backend communication.
  * hiveCloud.ts: Service for hive data operations.
  * farmersApi.js: Service for farmer directory operations.
  * chatApi.js: Real-time communication service.
  * location.js: GPS and mapping utilities.
- /constants: Configuration (API URLs, styling tokens, etc.)
- /assets: Images, fonts, and static resources.

BACKEND (Root: ./backend)
-------------------------
The backend is a Node.js/Express server written in TypeScript.

- /src/controllers: Request handlers and business logic.
- /src/services: Deep logic for database operations (e.g., auth, sync).
- /src/routes: API endpoint definitions.
- /src/db: Database schema (PostgreSQL) and connection client (Supabase).
- /src/middlewares: Auth validation, error handling, and security.
- /src/validators: Data validation logic (Zod).
- API_DOCUMENTATION.md: Detailed REST endpoint specifications.


4. HOW IT WORKS
---------------
1. Authentication: Users login via phone number. The server sends an OTP, 
   which the user verifies to receive a JWT token.
2. Data Fetching: The app fetches farmer and hive data from the backend.
3. Offline Operations: When offline, the app stores changes locally.
4. Syncing: Once online, the app triggers a bulk sync. The server compares 
   'updated_at' timestamps to resolve any conflicts between local and 
   remote versions.
5. Media: Photos taken in the field are uploaded to cloud storage and 
   linked to the specific hive record for traceability.
